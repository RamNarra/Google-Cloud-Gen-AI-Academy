What Is an Agent — Prashanth's Definition
Definition
An application that uses a generative AI model to make decisions
Acts independently based on its own reasoning
Pursues a user-specified goal — not a fixed instruction sequence
You tell it what to achieve, not how
Three Properties
Flexible — handles edge cases it was never explicitly programmed for
Outcome-focused — goal sets the direction, agent picks the path
Proactive — can run on schedules and triggers, not just user input
When to Use Agents — and When Not To
Prashanth Was Explicit About Both
USE
Multi-step, goal-oriented workflows that require dynamic decision-making
USE
Personalised, adaptive systems (learning, customer experience, real-time business intelligence)
USE
Cybersecurity monitoring, triage, vulnerability detection — confirmed in Q&A
SKIP
Simple prompt → response tasks. Just use an API call.
SKIP
Q&A from a static knowledge base. Use a RAG architecture instead.
SKIP
Fully regulated environments requiring explainability of every decision. Use predictive AI (classifiers).
Honest Challenges Prashanth Named
Know These Before You Build
C1
Non-determinism — same input may not produce same output. LLMs are inherently probabilistic.
C2
Immature error handling — agents can enter unexpected failure states with no recovery path. Mindset shift required from traditional software.
C3
Security & privacy still evolving — prompt injection, tool manipulation, data leakage via agents are open problems.
C4
Hard to debug — you can see what an agent did, but tracing why the LLM chose that path is often impossible.

Key Q&A Answers
Direct Answers from Prashanth
Prerequisites?
Just basic programming. Python preferred but not mandatory. No ML/maths background needed.
Which database to use?
The curriculum uses AlloyDB + Cloud SQL (PostgreSQL-compatible). If you know Postgres, you're set.
Will Cloud Run cost money?
Free tier is generous (~1000+ calls/day free). Credits are provided. Development will cost you nothing.
Can agents be used in cybersecurity?
Yes, absolutely. Triage, monitoring, vulnerability checking. Google Cloud also has out-of-the-box security agents including MCP server protection.
How to pick and evaluate a model?
Start with Gemini Flash. Build your eval framework first (test cases with expected outputs). Then swap models and compare against evals. ADK has a built-in eval tab.
Agents in regulated environments?
Still possible. Use Google Cloud in-region deployment for data residency. Add human-in-the-loop for medical/legal. Use ADK workflow agents (not fully autonomous) for explainability requirements.
